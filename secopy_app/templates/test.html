Basics

Question 1: What is the recommended Python version?

It is always the latest stable version, because many vulnerabilities have been already patched in the latest releases.

Question 2: Why is recommended using venv (virtual environment)?

When using virtual environment, user has acces only to modules which are imported to particular venv, therefore other projects cannot effect each other. Even malicious modules cannot spread in such environment. It could be theoretically faster for interpreter, but we are talking about secure coding.

Question 3: What kind of module import is this?

Implicit import has been banned from Python since version 3, relative import contains character ., so it could be explicit or absolute. According to the guidelines it is absolute import. Explicit import is not entirely wrong, because it shows exactly what we want to import, but this test is made to test the guidelines.

Question 4: How should programmer import module?

The most secure way of coding is to not import at all, but this question asks why not if. If we are importing a module it is important to know exactly what methods we need and import only them. It lowers the risk of importing something malicious.

Question 5: What is typosquatting?

It is certainly the first answer, because typosquatting is kind of cyberattack, which uses user's lack of attention.

Standard library

Question 6: The best way how to handle input according to the guidelines?

Again, question asks how, so not having input is right, but not in this questions. The most secure way according to guidelines is whitelisting. Input must be validated, if not code becomes very vulnerable. Blacklisting cannot validate all of the possible inputs.

Question 7: What is so called "zip bomb"?

It is called zip, because it uses .zip file and bomb because extracting such file causes massive loss of memory or crash of app. There are zip bomb files in sizes of kiloBytes which need 3 GigaBytes of memory to extract.

Question 8: When to use if-else?

If user do not know which error could occur or works with files he should use try-catch block. If-else block is reguralry used for flow control which means that is beneficial to use it if there is a bigger chance for mistake to happen. With smaller chance try-except becomes more efficient, because one try-catch can cover many mistakes.

Question 9: Which of these methods was patched from Python 3.0 on wards?

Only method input. See the guidelines for more information.

Question 10: What attack can attacker create by using regular expression?

ReDoS (Regular expression denial of services). By creating input such as aaaaaaaaaaaaaaaaaaaaaaaaaaa! which can cause timeout when evil regex is present.

Web programming

Question 11: Which organisation makes list of 10 most occuring web app vulnerabilities?

Every developer should know that the answer is OWASP. The top 10 is one of the most vital list for web developer.

Question 12: What is the top 1 vulnerability in the 2021 list?

Broken Access Control. In 2017 it was Injection.

Question 13: How to prevent from cryptographic failures vulnerability?

Using open-source cryptographic algorithms is not good since the algorithms are public even for attackers. The best method is to use the latest and updated algorithms which are not known even for attackers. HTTP is not safe and not encrypting too.

Question 14: How to prevent from SQL injection vulnerability?

For SQL injection is enough 6 characters, so user has to validate the input thoroughly and before SQL query execution.

Question 15: What happens after theft of session ID?

When attacker steals user's identity he can do whatever he wants. Session can be stolen and it is used for such purposes as is sending money to attacker's account or buying stuff with user's credit card.
